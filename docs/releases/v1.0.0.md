# opencode-actions v1.0.0

First public release of `opencode-actions`.

## What ships in v1.0.0

- `setup-opencode`: install and cache OpenCode on Linux runners
- `run-opencode`: run OpenCode with optional retry support
- `github-run-opencode`: one-step wrapper for the common `opencode github run` workflow
- shell-based regression tests for installer, fallback, cache recovery, and retry flows
- example workflows for PR review and comment-triggered commands
- CI that lint-checks shell scripts and smoke-tests all three actions locally

## Notes

- Linux only in v1.0.0
- installation is currently installer-based via `https://opencode.ai/install`
- cache is best-effort and should not be treated as an exact version pin
- the comment-command example is restricted to trusted repository collaborators and skips fork PRs

## Recommended upgrade path

- Pin to `@v1` for compatible updates
- Pin to a full commit SHA if you need maximum supply-chain stability

## Example

```yaml
- name: Run OpenCode review
  uses: Svtter/opencode-actions/github-run-opencode@v1
  env:
    MODEL_NAME: zhipuai-coding-plan/glm-5.1
  with:
    github-token: ${{ secrets.GITHUB_TOKEN }}
    zhipu-api-key: ${{ secrets.ZHIPU_API_KEY }}
    opencode-go-api-key: ${{ secrets.OPENCODE_GO_API_KEY }}
```
