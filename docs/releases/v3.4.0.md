# opencode-actions v3.4.0

## What's Changed

### Breaking
- **`extra-env` now blocks sensitive key overrides by default**: entries in `extra-env` that override sensitive runtime variables (`GITHUB_TOKEN`, `MODEL`, `PROMPT`, `*_API_KEY`, etc.) or use the reserved `GITHUB_RUN_OPENCODE_` prefix now produce an `::error::` and exit 1 instead of a `::warning::`. To restore the previous behavior, set `extra-env-allow-sensitive: "true"`. (#174)

### Added
- **New `extra-env-allow-sensitive` input** (default `false`): when set to `true`, `extra-env` entries that override sensitive runtime variables are allowed with a `::warning::` instead of being blocked. Available on `review`, `feature-missing`, `spec-coverage`, `architect-review`, and `github-run-opencode`. (#174)
- **Bilingual (zh/en) language support**: new `language` input on all agent-based actions (`review`, `feature-missing`, `spec-coverage`, `github-run-opencode`, `architect-review`, `multi-review`). Default `zh` for Chinese, `en` for English. (#170)
- **`extra-env` and `extra-env-allow-sensitive` inputs on `github-run-opencode`**: previously missing, now aligned with other actions. (#174)

### Fixed
- Complete `SENSITIVE_ENV_KEYS` set: added `MINIMAX_API_KEY`, `XIAOMI_API_KEY`, `USE_GITHUB_TOKEN`, `GITEA_TOKEN`, and all `OPENCODE_*` internal config vars. (#171, #172)
