# opencode-actions v3.0.0

## What's Changed

### Added
- **New `architect-review` action**: architecture-level PR review action (#64)
- **New `multi-review` action**: multi-agent parallel code review with coordinator synthesis (#81)
  - Collapsible reviewer details in coordinator comment
  - Global session cleanup as safety net (#115)
  - Rewritten using OpenCode SDK for native integration (#103)
  - Replaced hand-written YAML parser with `js-yaml` (#111)
- **Gitea CI compatibility**: platform abstraction layer supporting both GitHub and Gitea (#119)

### Changed
- Default thinking-intensity changed from `max` to `high` (#121)
- Updated `setup-ci` skill: added `architect-review` and `multi-review` (#88, #91)
- Updated README with multi-review documentation (#109)

### Fixed
- Configure git identity before opencode runs to prevent commit failures
- Surface opencode errors to CI and catch model-not-found in fallback (#100)
- Deny bare git commands in read-only action permission config (#79)
- Ensure multi-review sessions are cleaned up on error paths (#114)
- Fix CI crashes and clean comment output in multi-review (#86)
- Improve comment noise filtering in multi-review (#87)
- Isolate reviewers in git worktrees (#90)
- Remove unused configPath parameter from `loadReviewers()` (#113)
- Align multi-review README example model provider name with code default (#112)

## New Contributors

This release introduces two new composite actions and a platform abstraction layer that enables Gitea CI usage alongside GitHub Actions.
